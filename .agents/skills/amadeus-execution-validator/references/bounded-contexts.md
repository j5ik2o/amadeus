# bounded-contexts validation

## 対象

- `.amadeus/domain/bounded-contexts.md`
- `.amadeus/intents/<intent-id>/domain/bounded-contexts.md`

## 必須見出し

全体の `bounded-contexts.md` は、次の見出しを持つ。

- `一覧`
- `コンテキスト間の依存`
- `パターン分類`

Intent 配下の `bounded-contexts.md` は、少なくとも次の見出しを持つ。

- `コンテキスト`
- `コンテキスト間の依存`

## 一覧表

全体の `一覧`、または Intent 配下の `コンテキスト` の表は、次の列を持つ。

- `識別子`
- `名前`
- `サブドメイン`
- `役割`
- `モデル`
- `契約`

`識別子` は `BCnnn` の形式にする。
同じ表の中で `識別子` を重複させない。

`モデル` は `bounded-contexts/<bounded-context-id>/models.md` を参照する。
`契約` は `bounded-contexts/<bounded-context-id>/contracts.md` を参照する。

## コンテキスト間の依存

`コンテキスト間の依存` の表は、次の列を持つ。

- `Downstream`
- `Upstream`
- `依存内容`
- `組織パターン`
- `統合パターン`
- `状態`

`Downstream` と `Upstream` には、同じ `bounded-contexts.md` の一覧に存在する境界づけられたコンテキスト ID、または `なし` を書く。

`依存内容` と `状態` は空欄にしない。

`Upstream` が `なし` の場合、`組織パターン` と `統合パターン` は `該当なし` にする。
この場合も、`状態` に理由を書く。

`Upstream` が `なし` でない場合、`組織パターン` は次のいずれかにする。

- `パートナーシップ`
- `別々の道`
- `順応者`
- `顧客／供給者`

`Upstream` が `なし` でない場合、`統合パターン` は次のいずれかにする。

- `共有カーネル`
- `巨大な泥団子`
- `公開ホストサービス（OHS）`
- `公表された言語（PL）`
- `腐敗防止層（ACL）`

## パターン分類

全体の `パターン分類` は、組織パターン4種類と統合パターン5種類をすべて列挙する。

組織パターンは次の4種類である。

- `パートナーシップ`
- `別々の道`
- `順応者`
- `顧客／供給者`

統合パターンは次の5種類である。

- `共有カーネル`
- `巨大な泥団子`
- `公開ホストサービス（OHS）`
- `公表された言語（PL）`
- `腐敗防止層（ACL）`
