# 契約

## 範囲

この文書は、`20260626-password-reset` インテントで守る契約を扱う。

契約は、事前条件、不変条件、事後条件で表す。
要求やユースケースに散在する条件を、実装と検証で見落とさないための入口として使う。

## 事前条件

| 識別子 | 事前条件 | 根拠 |
|---|---|---|
| PRE001 | パスワード再設定要求は、利用者がアカウントに紐づく識別子を入力できる時に開始できる。 | R001, UC001 |
| PRE002 | 再設定トークン発行は、入力された識別子に対応するアカウントが存在する場合だけ成立する。 | R001, UC001 |
| PRE003 | 認証情報更新は、利用者が再設定トークンと新しい認証情報を入力できる時に開始できる。 | R002, UC002 |
| PRE004 | 認証情報更新は、再設定トークンが有効期限内、未使用、対象アカウントに紐づく場合だけ成立する。 | R002, UC002 |

## 不変条件

| 識別子 | 不変条件 | 根拠 |
|---|---|---|
| INV001 | 再設定トークンは単回利用である。 | R002, UC002 |
| INV002 | 再設定トークンは有効期限を過ぎると使用できない。 | R001, R002, UC002 |
| INV003 | 再設定トークンは、紐づくアカウント以外の認証情報を変更できない。 | R002, UC002 |
| INV004 | 認証情報更新後、対象の再設定トークンは消費済みになる。 | R002, UC002 |
| INV005 | 使用済み再設定トークンは再利用できない。 | R002, UC002 |
| INV006 | 存在しないアカウントに対するパスワード再設定要求でも、アカウントの存在有無を推測できる応答を返さない。 | R001, UC001 |

## 事後条件

| 識別子 | 事後条件 | 根拠 |
|---|---|---|
| POST001 | 対応するアカウントが存在する場合、期限付きの再設定トークンが発行されている。 | R001, UC001 |
| POST002 | パスワード再設定要求の応答から、アカウントの存在有無を推測できない。 | R001, UC001 |
| POST003 | 有効な再設定トークンを使った場合、アカウントの認証情報が更新されている。 | R002, UC002 |
| POST004 | 認証情報更新に使われた再設定トークンは消費済みである。 | R002, UC002 |
| POST005 | 期限切れ、使用済み、不正な再設定トークンでは、認証情報は更新されない。 | R002, UC002 |

## 整合性境界

アカウントは、認証情報と再設定トークンの整合性を守る候補である。

認証情報更新と再設定トークン消費は、片方だけが成立した状態を残さない。
