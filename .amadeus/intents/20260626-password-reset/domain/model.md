# ドメインモデル

## 範囲

このモデルは、`20260626-password-reset` インテントで扱う概念関係を定義する。

全体の境界づけられたコンテキストと共通概念は、[../../../domain-model.md](../../../domain-model.md) を参照する。

このインテントの境界づけられたコンテキスト、外部境界、Unit 分割の判断基準は、[bounded-context.md](bounded-context.md) を参照する。

## 概念

| 概念 | 説明 | 参照 |
|---|---|---|
| 利用者 | ログインできない状態で、パスワード再設定を求める人である。 | ACT001 |
| アカウント | 利用者に紐づき、認証情報と再設定トークンの整合性を守る対象である。 | R001, R002 |
| アカウント識別子 | 利用者がパスワード再設定要求で入力する、アカウントを探すための識別子である。 | UC001 |
| 認証情報 | ログイン認証に使う秘密情報である。 | R002, UC002 |
| 再設定トークン | 認証情報更新を許可する、期限付きで単回利用の証明である。 | R001, R002 |
| 再設定手段 | 利用者へ再設定トークンを届けるための手段である。 | UC001 |
| メール配信サービス | 再設定手段の送信依頼を受ける外部システムである。 | EXT001 |

## 関係

- 利用者はアカウントに紐づく。
- アカウントは認証情報を持つ。
- 利用者はアカウント識別子を使ってパスワード再設定を要求する。
- パスワード再設定要求は、対応するアカウントが存在する場合に再設定トークンを発行する。
- 再設定トークンはアカウントと有効期限に紐づく。
- 再設定トークンは認証情報更新に使われる。
- 認証情報更新は再設定トークンを消費済みにする。
- メール配信サービスは、再設定手段を利用者へ届けるために使われる。

## ライフサイクル

```text
パスワード再設定要求済み
  -> 再設定トークン発行済み
  -> 再設定トークン検証済み
  -> 認証情報更新済み
  -> 再設定トークン消費済み
```

## 集約候補

### アカウント

アカウントは、認証情報、再設定トークン、再設定トークン消費の整合性を守る候補である。

再設定トークンの発行、検証、消費は、対象アカウントの認証情報更新と矛盾しない形で扱う。
