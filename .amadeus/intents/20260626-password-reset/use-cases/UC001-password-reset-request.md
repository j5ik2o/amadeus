# UC001：利用者がパスワード再設定を要求する

## 要求

- R001

## ストーリー

- S001

## アクター

- ACT001

## 外部システム

- EXT001

## システム境界

- 外側: ACT001, EXT001
- 内側: システム
- 境界: パスワード再設定要求

## 事前条件

- 利用者はログインできない状態にある。
- 利用者はアカウントに紐づく識別子を入力できる。

## 基本フロー

1. 利用者は、アカウントに紐づく識別子を入力する。
2. システムは、入力された識別子に対応するアカウントを確認する。
3. システムは、再設定トークンを発行する。
4. システムは、メール配信サービスへ再設定手段の送信を依頼する。
5. システムは、アカウントの存在有無を推測できない応答を利用者へ返す。

## 代替フロー

- 対応するアカウントが存在しない場合でも、システムはアカウントの存在有無を推測できない応答を返す。
- メール配信サービスへの依頼に失敗した場合、システムは失敗を記録し、利用者へアカウントの存在有無を推測できない応答を返す。

## 事後条件

- 対応するアカウントが存在する場合、期限付きの再設定トークンが発行されている。
- 利用者への応答から、アカウントの存在有無を推測できない。

## BCE候補

| 種別 | 候補 | 根拠 |
|---|---|---|
| 境界 | パスワード再設定要求 | 利用者がアカウントに紐づく識別子を入力する接点 |
| 境界 | 再設定手段送信依頼 | システムがメール配信サービスへ送信を依頼する接点 |
| 制御 | 再設定トークン発行 | アカウント確認、再設定トークン発行、送信依頼を調整する処理 |
| エンティティ | アカウント | 認証情報と再設定トークンの整合性を持つ概念 |
| エンティティ | 再設定トークン | 期限付きで単回利用される概念 |

## 責務候補

| 候補 | 責務 |
|---|---|
| パスワード再設定要求 | 利用者からアカウントに紐づく識別子を受け取る |
| 再設定手段送信依頼 | メール配信サービスへ再設定手段の送信を依頼する |
| 再設定トークン発行 | アカウント確認、再設定トークン発行、送信依頼を調整する |
| アカウント | 認証情報と再設定トークンの整合性を保つ |
| 再設定トークン | 有効期限と単回利用状態を保持する |
