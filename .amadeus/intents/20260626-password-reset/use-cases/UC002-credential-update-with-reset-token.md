# UC002：利用者が再設定トークンで認証情報を更新する

## 要求

- R002

## ストーリー

- S002

## アクター

- ACT001

## 外部システム

- なし

## システム境界

- 外側: ACT001
- 内側: システム
- 境界: 認証情報更新

## 事前条件

- 利用者は再設定トークンを受け取っている。
- 利用者は新しい認証情報を入力できる。

## 基本フロー

1. 利用者は、再設定トークンと新しい認証情報を入力する。
2. システムは、再設定トークンが有効期限内で未使用であることを確認する。
3. システムは、再設定トークンに紐づくアカウントを確認する。
4. システムは、アカウントの認証情報を更新する。
5. システムは、再設定トークンを消費済みにする。
6. システムは、認証情報更新の完了を利用者へ返す。

## 代替フロー

- 再設定トークンが期限切れの場合、システムは認証情報を更新しない。
- 再設定トークンが使用済みの場合、システムは認証情報を更新しない。
- 再設定トークンが不正な場合、システムは認証情報を更新しない。

## 事後条件

- 有効な再設定トークンを使った場合、アカウントの認証情報が更新されている。
- 認証情報更新に使われた再設定トークンは再利用できない。

## BCE候補

| 種別 | 候補 | 根拠 |
|---|---|---|
| 境界 | 認証情報更新要求 | 利用者が再設定トークンと新しい認証情報を入力する接点 |
| 制御 | 再設定トークン検証 | 有効期限、使用状態、対象アカウントを確認する処理 |
| 制御 | 認証情報更新調整 | 認証情報更新と再設定トークン消費を調整する処理 |
| エンティティ | アカウント | 認証情報を持つ概念 |
| エンティティ | 認証情報 | ログイン認証に使う秘密情報 |
| エンティティ | 再設定トークン | 認証情報更新に使われ、消費後に再利用できない概念 |

## 責務候補

| 候補 | 責務 |
|---|---|
| 認証情報更新要求 | 利用者から再設定トークンと新しい認証情報を受け取る |
| 再設定トークン検証 | 再設定トークンの有効期限、使用状態、対象アカウントを確認する |
| 認証情報更新調整 | 認証情報更新と再設定トークン消費を調整する |
| アカウント | 認証情報を保持し、対象アカウントの整合性を保つ |
| 認証情報 | ログイン認証に使う秘密情報を表す |
| 再設定トークン | 有効期限、使用状態、対象アカウントとの紐づきを保持する |
