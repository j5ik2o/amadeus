# D002: レビュー支援契約の採用判断

## 背景

skill 本文変更のレビュー負荷を下げる対応候補には、skill 本文変更ごとの behavioral eval を必須化する自動化案と、レビュー支援の運用契約案があった。

## 判断

behavioral eval の必須化を非採用にする。

代わりに、次の3点をレビュー支援の運用契約として採用する。

1. 変更種別「skill 変更」の必須条件に、挙動差分の要約を追加する。
2. skill 変更 PR の作成前に skill-forge による確認を実施し、結果を PR 説明に記録する。
3. skill 本文変更と他の変更種別を同一 PR に混ぜない粒度制約を追加する。

## 理由

skill 本文は散文であるため、先に失敗するテストの型が使えない。
behavioral eval の必須化は作成コストが高く、自己開発の流量を止める。
採用した3点は、既存の PR 説明の慣行と README の推奨を必須条件へ格上げする形であり、書く側のコスト増が小さい。

## 影響

scope の SC-OUT-001（behavioral eval の必須化）を対象外として固定する。
Inception では、採用した運用契約3点を要求として分解する。
