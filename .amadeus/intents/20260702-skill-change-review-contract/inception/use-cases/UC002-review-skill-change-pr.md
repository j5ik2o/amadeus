# UC002 skill 変更 PR をレビューする

## ユースケース

Reviewer が skill 変更 PR をレビューするとき、PR 説明の記録から変更の影響と確認済みの観点を把握し、記録が不足する PR を差し戻す。

## アクター

- ACT003 Reviewer

## 外部システム

- EXT001 GitHub

## 事前条件

- PR 説明に記録すべき条件が `steering/policies.md` に定義されている。
- `development.md` と README の記述が policies と整合しており、Reviewer が参照する契約文書間に矛盾がない。

## 基本フロー

1. Reviewer は、PR 説明から挙動差分の3観点を読む。
2. Reviewer は、固定見出し「skill-forge 確認」から確認した観点と確認結果を読む。
3. Reviewer は、PR が skill 変更だけで構成されているか確認する。
4. Reviewer は、記録と変更差分を突き合わせて、変更の影響を判断する。

## 代替フロー

| 条件 | 扱い |
|---|---|
| 挙動差分要約または skill-forge 確認の記録が不足している。 | 記録不足として差し戻す。 |
| 例外の記録がないまま、他の変更種別が混ざっている。 | 分割を求めて差し戻す。 |
