# UC003 粒度制約の例外を承認する

## ユースケース

Maintainer が、粒度制約の例外を使った skill 変更 PR で、記録された理由と後続確認先から例外の妥当性を判断する。

## アクター

- ACT001 Maintainer

## 外部システム

- EXT001 GitHub

## 事前条件

- 例外の判定基準と記録の型が `steering/policies.md` に定義されている。

## 基本フロー

1. Maintainer は、例外を使った PR の説明から、理由と後続確認先を読む。
2. Maintainer は、不可分の判定基準（分割するとどちらかの PR 単独で検証が fail する）に照らして妥当性を判断する。
3. Maintainer は、妥当と判断した場合は merge 判断へ進み、後続確認先の追跡を残す。

## 代替フロー

| 条件 | 扱い |
|---|---|
| 理由または後続確認先が記録されていない。 | 記録を求めて差し戻す。 |
| 分割してもどちらの PR も単独で検証が pass すると判断した。 | PR の分割を求める。 |
