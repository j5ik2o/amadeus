# S001: Reviewer の影響把握

## ストーリー

Reviewer として、skill 変更 PR をレビューするとき、生の diff を読み解く前に、挙動差分要約と skill-forge 確認の記録から変更の影響と確認済みの観点を把握したい。

それにより、レビュー1件あたりの判断時間を減らし、skill 本文変更の影響の見落としを防ぎたい。

## 受け入れ条件

- PR 説明の固定の場所から、挙動差分の3観点（変わる判断、変わる成果物構造、影響する後続 phase）を読める。
- PR 説明の固定見出し「skill-forge 確認」から、確認した観点と確認結果を読める。
- 記録が不足する PR を差し戻す基準を、policies から確認できる。

## 根拠

- R001、R002。
- `ideation/ideation.md` の体制で、Reviewer の関心を「挙動差分要約から skill 変更の影響を判断し、粒度制約が守られているかを確認する」と定義している。
