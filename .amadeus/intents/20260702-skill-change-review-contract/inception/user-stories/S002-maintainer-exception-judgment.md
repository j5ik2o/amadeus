# S002: Maintainer の例外妥当性判断

## ストーリー

Maintainer として、粒度制約の例外が使われた skill 変更 PR で、例外の理由と後続確認先の記録から妥当性を判断したい。

それにより、粒度制約の形骸化を防ぎつつ、不可分な変更を不自然に分割させたくない。

## 受け入れ条件

- 例外の判定基準（分割するとどちらかの PR 単独で検証が fail する不可分な場合）を policies から確認できる。
- 例外を使った PR の説明から、理由と後続確認先を読める。

## 根拠

- R003。
- `steering/actors.md` で、Maintainer の関心を「変更範囲、検証結果、次回 stage0 採用可否を判断できること」と定義している。
