# R001 挙動差分要約

## 要求

変更種別「skill 変更」の PR 説明で、挙動差分の要約を固定の3観点から確認できる。

## 背景

skill 本文の変更は散文であるため、テストでは落ちず、レビュアーは生の diff から影響を推測するしかない。

レビュー判断に必要な情報を PR 説明側で構造化することで、レビュー1件あたりの判断時間を減らす。

## 受け入れ条件

- `steering/policies.md` の変更種別「skill 変更」の必須条件に、挙動差分要約の記録が含まれている。
- 挙動差分要約の最低ラインが、固定の3観点（変わる判断、変わる成果物構造、影響する後続 phase）として定義されている。
- 3観点に加えて、補足を自由記述で追加できることが読み取れる。
