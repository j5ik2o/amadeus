# D003: Unit と Bolt の粒度判断

## 背景

- Issue #243 は、参照リンク化方針、GitHub permalink 方針、適用範囲、validator の扱いを同時に扱う。
- これらを1つの Unit にまとめると、方針契約と検出境界が混ざる。
- 参照リンク化対象と適用範囲を定義しないまま validator 検出を作ると、誤検出や対象外の扱いが曖昧になる。

## 判断

- Unit は、参照リンク化方針契約を扱う `U001` と、検出境界を扱う `U002` に分ける。
- Bolt は、参照リンク化規則、成果物適用範囲、検出境界の3つに分ける。

## 理由

- `U001` は、参照リンク化対象、リンク先規則、適用成果物範囲を扱う価値単位である。
- `U002` は、validator、eval、人間判断の分担を扱う価値単位であり、`U001` の方針を前提にする。
- `B001`、`B002`、`B003` に分けることで、Construction の Task Generation で方針、適用、検出を別々に扱える。

## 影響

- Construction では、`B001` の完了後に `B002`、その後に `B003` を進める。
- `B003` は、validator と eval の具体化を扱うため、TDD スタイルの検証追加を前提にする。
- Unit と Bolt が完全な 1:1 にならないため、粒度不足の例外理由は不要である。
