# ドメインモデル

## 境界づけられたコンテキスト

### 認証アクセス

利用者、アカウント、認証情報、再設定トークンを扱う。

認証アクセスは、利用者が本人であることを確認し、アカウントに紐づく認証情報を管理する。

## 概念間の関係

- 利用者はアカウントに紐づく。
- アカウントは認証情報を持つ。
- アカウントはパスワード再設定を要求できる。
- パスワード再設定は再設定トークンを発行する。
- 再設定トークンはアカウントと有効期限に紐づく。

## 不変条件

- 再設定トークンは単回利用である。
- 再設定トークンは有効期限を過ぎると使用できない。
- 再設定トークンは対象アカウント以外の認証情報を変更できない。
- パスワード再設定完了後、使用済み再設定トークンは再利用できない。

## ライフサイクル

```
パスワード再設定要求済み
  -> 再設定トークン発行済み
  -> 再設定トークン検証済み
  -> 認証情報更新済み
  -> 再設定トークン消費済み
```

## 集約候補

### アカウント

アカウントは認証情報とパスワード再設定の整合性を守る候補である。

再設定トークンの発行、検証、消費はアカウントに紐づく操作として扱う。
