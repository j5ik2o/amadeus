# S001: 最初の Intent 候補選択

## ストーリー

- Amadeus 利用者として、大きな入力テーマから整理された Discovery Brief を読み、最初に進める Intent 候補を選べるようにしたい。
- それにより、入力テーマを無理に1つの Intent にせず、後続 phase へ進める候補を確認できる。

## 受け入れ条件

- Discovery Brief の入力テーマ、判定、判定理由を確認できる。
- Intent 候補と候補判断を確認できる。
- 最初に進める Intent 候補を確認できる。

## 根拠

- `ideation.md` の「体制」に、Amadeus 利用者が最初に進める候補を確認したいことが記録されている。
- `mocks/initial-confirmation.puml` に、Discovery Brief 確認カードの初期モックがある。
