# Frontend Components

## 目的

注文作成 Unit の UI 構成候補を Task 生成の根拠として記録する。

## UI 構成

| 識別子 | 名前 | 責務 | 状態 |
|---|---|---|---|
| UI001 | 注文作成結果表示 | 作成された注文を顧客へ示す。 | proposed |
| UI002 | 注文作成エラー表示 | 注文作成へ進めない理由を顧客へ示す。 | proposed |

## 状態

- 作成済み注文を表示できる。
- 不足条件がある場合は、注文作成へ進めない理由を表示できる。

## 未確認事項

- 表示する注文番号、注文状態、永続化済みを示す文言は未確認である。
